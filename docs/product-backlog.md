## 1. 项目概述

### 1.1 项目背景

本系统是一个面向企业的项目全流程管理 CRM 系统，覆盖从客户获取、商机跟进、合同签订、项目执行到项目验收的完整业务流程。系统旨在提升企业销售管理和项目交付效率，实现业务流程数字化管理。

### 1.2 现有基础

- **后端框架**: Express + TypeScript + TypeORM + SQLite
- **前端框架**: React + TypeScript + Vite + Ant Design
- **已实现功能**: 用户认证、基础路由、实体模型、部分 API 接口

### 1.3 目标用户

| 角色 | 职责 | 使用频率 |
|------|------|----------|
| 系统管理员 | 系统配置、用户管理、权限管理 | 低频 |
| 销售经理 | 客户管理、商机跟进、合同签订 | 高频 |
| 项目经理 | 项目立项、进度管理、资源协调 | 高频 |
| 财务人员 | 收款管理、开票管理、成本核算 | 中频 |
| 审批人员 | 各类业务审批 | 中频 |

## 3. 模块详细需求

### 3.1 客户管理模块

#### 3.1.1 模块概述

客户管理是 CRM 系统的基础模块，用于维护客户基本信息、联系人、跟进记录等数据，支撑后续商机、合同、项目等业务开展。

#### 3.1.2 用户故事

##### US-001: 客户信息录入

**用户故事**:
作为销售经理，我希望能够录入新客户信息，以便建立客户档案并开展后续业务跟进。

**优先级**: P0

**验收标准(AC)**:
- [ ] 支持录入客户名称、统一社会信用代码、所属行业、客户级别、客户状态等基本信息
- [ ] 客户名称必须唯一，系统自动校验重复
- [ ] 支持上传客户营业执照等附件
- [ ] 录入后自动生成客户编号，格式: CUST-YYYY-XXXXX
- [ ] 记录创建人和创建时间
- [ ] 支持保存草稿和提交正式两种模式

**估算**: 5 Story Points

---

##### US-002: 客户信息查询与筛选

**用户故事**:
作为销售经理，我希望能够按照多种条件查询和筛选客户，以便快速定位目标客户群体。

**优先级**: P0

**验收标准(AC)**:
- [ ] 支持按客户名称、编号模糊搜索
- [ ] 支持按客户级别(A/B/C级)、状态(潜在/正式/流失)、所属行业筛选
- [ ] 支持按客户经理、创建时间范围筛选
- [ ] 支持组合条件查询
- [ ] 查询结果支持分页展示，默认每页 20 条
- [ ] 支持将常用查询条件保存为快捷筛选

**估算**: 3 Story Points

---

##### US-003: 联系人管理

**用户故事**:
作为销售经理，我希望能够为每个客户维护多个联系人信息，以便记录客户方的关键对接人。

**优先级**: P0

- [ ] 联系人信息包括: 姓名、性别、职务、部门、电话、邮箱、微信、是否决策人
- [ ] 支持设置默认联系人
- [ ] 支持标记联系人的角色(决策人/使用人/技术负责人/财务人员)
- [ ] 联系人手机号必须唯一性校验
- [ ] 支持查看联系人历史跟进记录

**估算**: 5 Story Points

---

##### US-004: 客户跟进记录

**用户故事**:
作为销售经理，我希望能够记录每次与客户的沟通跟进情况，以便追踪客户关系发展过程。

**优先级**: P1

**验收标准(AC)**:
- [ ] 支持记录跟进方式(电话/邮件/拜访/会议/其他)
- [ ] 支持记录跟进时间、参与人员、沟通内容、下次跟进计划
- [ ] 支持关联联系人
- [ ] 跟进记录按时间倒序展示
- [ ] 支持上传沟通相关附件
- [ ] 支持@相关同事进行协作
- [ ] 下次跟进时间到期前自动提醒

**估算**: 5 Story Points

---

##### US-005: 客户级别升降级

**用户故事**:
作为销售经理，我希望能够调整客户级别，以便根据客户价值和合作深度进行分类管理。

**优先级**: P1

### 3.2 商机管理模块

#### 3.2.1 模块概述

商机管理模块用于追踪销售机会从发现到赢单或输单的完整生命周期，帮助销售团队预测业绩和管理销售漏斗。

#### 3.2.2 用户故事

##### US-006: 商机创建

**用户故事**:
作为销售经理，我希望能够创建新的销售商机，以便记录潜在的销售机会。

**优先级**: P0

**验收标准(AC)**:
- [ ] 商机必须关联已存在的客户
- [ ] 录入商机名称、商机来源、预计金额、预计成交日期
- [ ] 选择商机阶段(初步接触/需求确认/方案报价/商务谈判/即将成交)
- [ ] 选择商机级别(A/B/C 级，A 级为重要商机)
- [ ] 自动计算商机赢率(根据阶段自动映射: 初步接触 10%、需求确认 30%、方案报价 50%、商务谈判 70%、即将成交 90%)
- [ ] 商机编号自动生成，格式: OPP-YYYY-XXXXX
- [ ] 记录商机负责人和创建时间

**估算**: 5 Story Points

---

##### US-007: 商机阶段推进

**用户故事**:
作为销售经理，我希望能够推进商机的销售阶段，以便反映销售进展。

**优先级**: P0

**验收标准(AC)**:
- [ ] 支持商机阶段向前推进和向后回退
- [ ] 阶段变更需要填写变更说明
- [ ] 推进到"方案报价"阶段需要上传方案文档
- [ ] 推进到"即将成交"阶段需要填写竞争情况分析
- [ ] 阶段变更历史可追溯
- [ ] 阶段停留超过设定天数自动预警(如商务谈判阶段超过 30 天)

**估算**: 5 Story Points

---

##### US-008: 商机赢单/输单处理

**用户故事**:
作为销售经理，我希望能够标记商机的赢单或输单结果，以便结束商机生命周期。

**优先级**: P0

**验收标准(AC)**:
- [ ] 支持标记商机为"赢单"或"输单"
- [ ] 赢单时需要关联新创建的合同
- [ ] 输单时需要选择输单原因(价格/产品功能/服务/竞争对手/客户延期/其他)
- [ ] 输单时填写详细输单分析
- [ ] 赢单/输单后商机状态变为"已关闭"
- [ ] 已关闭商机不允许修改基本信息

**估算**: 5 Story Points

---

##### US-009: 销售漏斗分析

**用户故事**:
作为销售总监，我希望能够查看销售漏斗分析报表，以便了解整体销售状况和预测业绩。

**优先级**: P1

**验收标准(AC)**:
- [ ] 按商机阶段展示各阶段的商机数量和金额
- [ ] 计算各阶段转化率
- [ ] 支持按时间段筛选(本月/本季度/本年度/自定义)
- [ ] 支持按销售团队/个人筛选
- [ ] 展示预计成交金额(各阶段金额 x 赢率)
- [ ] 支持导出报表为 Excel

**估算**: 5 Story Points

---

##### US-010: 商机变更审批

**用户故事**:
作为销售经理，我希望重大商机变更能够发起审批流程，以便确保重要变更得到管理层确认。

**优先级**: P1

**验收标准(AC)**:
- [ ] 预计金额增加超过 50%需要发起变更审批
- [ ] 预计成交日期延期超过 30 天需要发起变更审批
- [ ] 审批流程包括: 提交人→部门经理→销售总监
- [ ] 审批通过后才生效变更
- [ ] 审批驳回需要重新提交或取消变更

**估算**: 8 Story Points

合同管理模块用于管理从合同起草、审批、签订到执行的完整生命周期，确保合同流程规范化和风险可控。

#### 3.3.2 用户故事

##### US-011: 合同创建与起草

**用户故事**:
作为销售经理，我希望能够基于赢单商机创建合同，以便启动合同签订流程。

**优先级**: P0

**验收标准(AC)**:
- [ ] 基于赢单商机一键创建合同，自动带入客户信息、金额等
- [ ] 支持选择合同类型(销售合同/服务合同/混合合同)
- [ ] 支持上传合同附件(Word/PDF)
- [ ] 录入合同金额、付款方式、签订日期、交付日期
- [ ] 支持添加合同条款备注
- [ ] 合同编号自动生成，格式: CONT-YYYY-XXXXX
- [ ] 合同状态初始为"草稿"

**估算**: 5 Story Points

---

##### US-012: 合同审批流程

**用户故事**:
作为销售经理，我希望合同提交后能够按配置流程审批，以便确保合同合规性。

**优先级**: P0

**验收标准(AC)**:
- [ ] 合同提交审批后状态变为"审批中"
- [ ] 根据合同金额自动选择审批流程:
  - 金额 < 10 万: 部门经理审批
  - 10 万 <= 金额 < 50 万: 部门经理→财务经理审批
  - 金额 >= 50 万: 部门经理→财务经理→总经理审批
- [ ] 每个审批节点可批准或驳回
- [ ] 驳回后返回起草人修改
- [ ] 审批通过后状态变为"待签署"
- [ ] 审批记录完整保存

**估算**: 8 Story Points

---

- [ ] 支持合同原件归档管理

**估算**: 5 Story Points

### 6.2 状态定义

#### 客户状态
- 潜在客户: 初步接触，尚未建立正式合作关系
- 正式客户: 已签订合同，建立正式合作关系
- 流失客户: 长期无业务往来或明确终止合作

#### 商机状态
- 进行中: 商机正在跟进中
- 赢单: 商机转化为合同
- 输单: 商机未成功
- 搁置: 暂时搁置跟进

#### 合同状态
- 草稿: 合同起草中
- 审批中: 合同正在审批流程中
- 待签署: 审批通过，等待双方签署
- 执行中: 合同已签署生效，正在执行
- 已完成: 合同义务履行完毕
- 已终止: 合同提前终止

#### 项目状态
- 立项审批中: 项目正在立项审批
- 执行中: 项目正在执行
- 验收中: 项目正在验收
- 已验收: 项目验收完成
- 已终止: 项目提前终止
